<FAQ title="Frequently Asked Questions">
  <FAQItem question="Is ShipSite free?">
    Yes. ShipSite is open source under the MIT License. Free to use for personal and commercial projects.
  </FAQItem>
  <FAQItem question="Do I need to know React?">
    No. You write MDX files using pre-built components. Drop in a Hero, add some Features, configure your shipsite.json — no React knowledge needed.
  </FAQItem>
  <FAQItem question="How does AI integration work?">
    ShipSite uses structured files (JSON config + MDX with typed components) that AI agents can read and write directly. Point an AI at your project and it can generate pages, modify config, and create blog posts — all through standard file operations.
  </FAQItem>
  <FAQItem question="Is the output SEO-optimized?">
    Yes. Every page is statically generated with proper meta tags, Open Graph data, canonical URLs, hreflang tags, and an auto-generated sitemap. Perfect for search engines.
  </FAQItem>
  <FAQItem question="Can I add custom components?">
    ShipSite ships with 30+ components that cover most marketing website needs. For custom components, you can extend the MDX component registry in your project.
  </FAQItem>
  <FAQItem question="Where can I deploy?">
    Anywhere that serves static files. Vercel, Cloudflare Pages, Netlify, AWS S3, or your own server. ShipSite generates standard Next.js static output.
  </FAQItem>
</FAQ>
